# Trus MVP — Stage 2b: Input Surfaces Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Contracts below are binding; the controller's dispatch carries drift notes.

**Goal:** Ship the brief's must-have input story — entry-as-interview (R-100), voice rambling (R-201–204), sketch-to-module (R-221–223), and the prescriptive idea-generation package (R-301–305) — plus the Stage-2a triaged backlog, so the product's front door matches the pitch: "tell me what's on your mind" → clarify → plan → build.

**Architecture:** Everything extends existing seams. Interview = the existing clarifying-question channel made genuinely multi-turn + a `plan` field on proposals. Voice = a new `/api/transcribe` route behind the same pluggable-provider pattern as LLM/vision (`TRUS_STT_*` env, OpenAI-compatible `/v1/audio/transcriptions`; unset → honest 422). Sketch = a canvas stroke overlay whose snap exports a PNG into the EXISTING image path with a sketch-tuned prompt. Suggestions = per-owner query over `gen_cache`/`messages` (R-903-scoped). Entry screen replaces IntroSplash as a true input door.

**Tech Stack:** unchanged. No new heavy deps (no bundled whisper — STT is a configured endpoint; sketch uses raw canvas 2D).
