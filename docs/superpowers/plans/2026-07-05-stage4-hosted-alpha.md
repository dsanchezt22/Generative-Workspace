### Task 1: Generate-route rate limiting + per-user cost surfacing (R-1202 completion, cost ceiling)
Backend: reuse Stage-3's `_RateLimiter` (now thread-safe) on the 5 LLM handlers in `routes/modules.py` (generate, preview, generate_from_file, refine, insights) — a shared per-owner generation limiter (default ≤`TRUS_GEN_RATE_MAX`=30 / `TRUS_GEN_RATE_WINDOW`=300s) → 429 honest message when exceeded (transcribe/live already limited; this closes the last unmetered spend surface the Stage-1 audit flagged). Add an OPTIONAL per-owner DAILY COST CAP: `db.owner_cost_today(owner) -> {tokens_in, tokens_out, cost_usd}` from `gen_events` (cost = tokens × `TRUS_TOKEN_COST_*`/1000; 0 rates → cost 0, tokens still shown); if `TRUS_DAILY_COST_CAP_USD` set and the owner is over it, the 5 handlers 429 "daily budget reached". Extend `/api/ops/summary` (already token-gated) with per-user cost/token rollup (reuse `last_seen_by_user`/`gen_stats` shape). TDD: 31st generate in-window → 429; over-cap owner → 429; ops shows per-user cost; cap unset → never blocks; rates 0 → cost 0 but tokens present. Commit `feat(backend): generate-route rate limit + per-owner cost cap & ops cost rollup (R-1202)`.

### Task 2: Touch-viable mobile (R-1304)
Frontend: the canvas uses unified pointer events + `touch-none` (pan/drag/module-move should already work on touch — VERIFY by reading the pointer handlers, note any mouse-only assumption). Add: (a) **pinch-zoom** — two-pointer gesture on the canvas (track two active pointers, zoom by the distance ratio toward the midpoint, reuse the existing zoom clamp; pointercancel/lift ends it); this is the one genuinely-missing touch capability (zoom is currently wheel+buttons only). (b) **viewport meta** in `app/layout.tsx` (`width=device-width, initial-scale=1`; do NOT set maximum-scale=1 — that blocks accessibility zoom; the canvas handles its own pinch, the rest of the page should allow browser zoom). (c) **Responsive legibility at 375px** — the right-hand panels (Profile/Snapshots/Archived/Conversation), the prompt bar, the entry screen, and module cards must not overflow/clip or force horizontal PAGE scroll at 375px wide; panels become full-width or a bottom sheet on narrow viewports; the prompt bar stays reachable. (d) Voice (MediaRecorder from 2b) already feature-detects — verify the mic affordance is reachable+sized for touch. Extract any pure gesture math (pinch distance/scale) to a lib + vitest. Manual-trace at 375px (document); R-1305 must survive the responsive changes. AC (R-1304): at 375px on mobile Safari/Chrome shapes — pinch-zoom + one-finger pan work, a module drag-repositions by touch, no clipped text / no horizontal page scroll, tap targets reachable. Commit `feat(frontend): touch-viable mobile — pinch-zoom, responsive panels, 375px legibility (R-1304)`.

### Task 3: Accessibility floor — keyboard + dialog semantics (R-1306)
Frontend: (a) **Canvas/module keyboard access** — a module is focusable (tabIndex), Enter/Space opens its detail/edit, arrow keys nudge a focused module's position (with a step; persist via the saver), Delete archives (with the existing confirm). A visible focus ring (ethos-consistent). (b) **Dialog semantics on the remaining overlays** — audit ConversationPanel, DetailView, CommandPalette, Inspector, and any other overlay for `role=dialog`/`aria-modal`, Escape-to-close, focus-into-on-open + restore-on-close, and a focus trap (ConfirmDialog/EntryScreen already have the pattern — reuse it; the Stage-2/3 reviews flagged these as the known a11y debt). (c) **Skip-to-content** + landmark roles (main/nav/complementary) so a screen-reader/keyboard user can reach the canvas without tabbing the whole sidebar. Manual-trace each overlay (Escape, Tab-trap, focus-restore); R-1306 is the AC. Note: full WCAG is NOT the bar — "keyboard-reachable, dialogs behave, focus never lost" is. Commit `feat(frontend): accessibility floor — keyboard canvas nav + dialog semantics on all overlays (R-1306)`.

### Task 4: Backup + restore mechanism (R-1106)
Backend/ops: (a) `python -m src.backup` CLI — checkpoints WAL (`PRAGMA wal_checkpoint(TRUNCATE)`) then copies `TRUS_DB_PATH` to a timestamped file under a `TRUS_BACKUP_DIR` (default `/data/backups`), keeping the last N (default 7); a `restore <file>` subcommand that swaps a backup in (with a safety copy of the current). Zero-dep stdlib. (b) `deploy/BACKUP.md` runbook — how to schedule it (cron/fly machine), the RPO≤24h target, and the "exercise a restore once before the alpha" step (a checklist item for the operator). TDD: backup creates a timestamped copy that opens as a valid DB with the same rows; retention prunes to N; restore swaps and preserves a distinctive row; WAL-checkpoint doesn't corrupt an in-use DB (backup a DB with pending WAL). Commit `feat(ops): SQLite backup+restore CLI and runbook (R-1106)`.

### Task 5: Profile accretion completion — prompt + activity (R-802 residue)
Backend: the R-802 spec names "interview answers, prompts, AND workspace activity" as accretion sources; Stage 3 shipped interview-only. Add, on the confirmed `POST /api/modules` insert: a light **prompt-derived** fact when the originating prompt states a durable goal/preference (source="prompt") — heuristic like the interview tagger, bounded ≤1/insert, deduped against existing; and a **workspace-activity** signal (source="activity") — e.g. on first creating a tool of a recognizable domain, note "uses a {domain} tracker" (kept minimal + honest; nothing the user can't see/delete). Reuse `_accrete_profile_facts`/the profile store; same cap50/dedup/owner-scope. Keep it conservative — a couple of high-signal facts, not noise. TDD: a goal-stating prompt on insert → a source="prompt" fact; an activity signal → a source="activity" fact; still capped/deduped/owner-scoped; a plain non-goal prompt → no prompt-fact (no noise). Commit `feat(backend): prompt + activity profile accretion (R-802 completion)`.

### Task 6: Live-data backlog batch

(c) `useLiveValue` stops polling once it learns `disabled` (clearInterval) — no wasted budget when `TRUS_LIVE_DATA=off`; manual-trace. (d) GaugeField loading shimmer (match MetricField) instead of a blank center number; manual-trace. Commit `fix: live-cache eviction + structured disabled flag + stop-poll-when-off + gauge loading (R-701 hardening)`.
